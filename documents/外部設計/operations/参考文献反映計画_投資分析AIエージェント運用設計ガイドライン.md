# 参考文献反映計画（投資分析AIエージェント運用設計ガイドライン）

最終更新日: 2026-03-03
対象文献:
- `documents/参考文献/投資分析AIエージェント運用設計ガイドライン：Claude Codeによる次世代分析環境の構築.md`

## 1. 目的

- 文献で提案される運用実践を、既存の `alpha-mind` 設計へ実装可能な単位で落とし込む。
- 追加価値が高い項目を優先度順に定義し、次の設計更新・実装タスクへ接続する。

## 2. 文献から抽出した反映要件

1. 検証回数を増やすため、仮説実行の摩擦を最小化する。
2. 人間は「何を検証するか」を定義し、AIは実行を担う役割分担を固定する。
3. 情報収集は「構造的アノマリー」と「イベント/ノイズ」を分離して扱う。
4. 日本株データは、調整係数を用いた逆順再計算（look-ahead回避）を標準化する。
5. 財務指標（例: BPS）を価格調整と同じ係数で同期し、単位ミスマッチを防止する。
6. 既存戦略の改善（エンパワーメント）を探索優先度の上位に置く。
7. LLM/APIコストを「通常運用」と「週次集中検証」で分離管理する。
8. 収集トリガー（例: Xブックマーク）から自動分析・通知までを一気通貫にする。

## 3. 現状ギャップ評価

| 要件 | 現状 | 判定 | 補足 |
|---|---|---|---|
| 役割分担（人間=意志、AI=実行） | `agent-orchestrator` + `instruction_profiles` で基盤あり | 部分適合 | 「検証意図（Neta）」の入力項目が未定義 |
| 構造的アノマリー/ノイズ分離 | `insight-collector` の収集・要約仕様あり | 不足 | 出力に `So What` 判定軸が未定義 |
| 逆順再計算ロジック | J-Quants列取得仕様は定義済み | 不足 | 再計算アルゴリズムが明文化されていない |
| BPS等の単位同期 | `feature-engineering` で時点整合は定義済み | 不足 | 財務指標の調整同期ルールが未定義 |
| 既存戦略の改善優先 | 失敗知見DB・再検証導線は存在 | 部分適合 | 「新規探索 vs 改善」の配分ルールが未定義 |
| LLM/APIコスト分離管理 | 月次コスト閾値（60 USD）は存在 | 部分適合 | 週次集中検証枠やX API重量課金の上限運用が未定義 |
| 自動分析通知 | イベント駆動収集は定義済み | 不足 | ブックマーク由来トリガー/通知経路が未定義 |

## 4. 組み込み方針（優先度順）

### P0: 実装前に確定（今スプリント）

1. `data-collector` に調整済み価格再計算仕様を追加する。
- 逆順ソート -> 調整係数cumprod -> shift(1) -> 価格乗算/出来高除算を標準処理に固定。
- `market.collected` 成果物は「調整済み」と「生値」を識別可能にする。

2. `feature-engineering` に財務指標同期ルールを追加する。
- BPS/PBR計算時に価格側と同一の調整係数を適用。
- 株式分割を跨ぐ期間の単純 `forward fill` を禁止。

3. `insight-collector` に `So What` 判定を追加する。
- `signalClass`（`structural_anomaly` / `event_noise`）を保持。
- `soWhatScore`（0.0-1.0）を必須計算し、閾値未満は仮説化優先度を下げる。

4. 運用設計へ「コスト二層管理」を追加する。
- 通常運用（Pro想定）と週次集中検証（Max相当）を分離。
- X APIとYouTube APIの月次上限・遮断条件をRunbookに追加。

### P1: 運用自動化（次スプリント）

1. Xブックマーク連動の `insight.collect.requested` トリガーを追加する。
2. `insight.collected` からDiscord通知（要点 + 根拠URL）を追加する。
3. 通知対象は `signalClass=structural_anomaly` かつ `soWhatScore>=閾値` に限定する。

### P2: 拡張検討（将来）

1. クリプト専用ソース（Binance/Coinbase/Bybit/OKX等）は、現行のJP株MVPと分離して別サービス化を検討。
2. ROIが確認できるまで有料集約API導入は保留し、PoCで費用対効果を計測する。
