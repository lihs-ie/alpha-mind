# 条件付き自動昇格設計

最終更新日: 2026-02-28

## 1. 目的

- 仮説昇格のうち「条件付き自動昇格」の許可条件・禁止条件・監査要件を明文化する。
- インサイダー接触回避要件と、運用効率（自動化）の両立条件を定義する。

## 2. 適用範囲

- 対象ドメイン: `hypothesis`
- 対象状態遷移: `demo -> live`
- 対象インターフェース:
- OpenAPI: `POST /hypotheses/{identifier}/promote`
- OpenAPI: `PUT /hypotheses/{identifier}/mnpi-self-declaration`
- AsyncAPI: `hypothesis.demo.completed`, `hypothesis.promoted`

## 3. 基本方針

- 無条件の自動昇格は禁止する。
- 条件付き自動昇格は許可する。
- 個別株は手動昇格を必須とする。

## 4. 必須要件（運用要件）

1. MNPI自己申告
- 昇格時に `mnpiSelfDeclared=true` を必須記録する。

2. 取引先関連銘柄ブロック
- `partnerRestrictedSymbols` に該当する銘柄は自動昇格禁止とする。

3. 監査証跡
- 昇格判断時に `actionReasonCode` と `trace` を必須記録する。

## 5. 判定ルール

### 5.1 自動昇格を許可する条件

- `instrumentType=ETF`
- `insiderRisk=low`
- `promotable=true`
- `demoPeriodDays>=30`
- `requiresComplianceReview=false`
- `mnpiSelfDeclared=true`
- `symbol not in partnerRestrictedSymbols`

### 5.2 自動昇格を禁止する条件

- `instrumentType=STOCK`
- `insiderRisk in {medium, high}`
- `mnpiSelfDeclared=false` または未設定
- `symbol in partnerRestrictedSymbols`
- `requiresComplianceReview=true`
- `demoPeriodDays<30`

## 6. 昇格モード

| 条件 | 昇格モード | 実行経路 |
|---|---|---|
| 5.1をすべて満たす | `auto` | `hypothesis.demo.completed` 受信時に `hypothesis.promoted` 発行 |
| 上記以外で昇格可能 | `manual` | `POST /hypotheses/{identifier}/promote` |
| 昇格不可条件あり | `blocked` | `COMPLIANCE_*` または `OPERATION_NOT_ALLOWED` で拒否 |

## 7. 契約マッピング

### 7.1 OpenAPI

- `HypothesisDecisionRequest`
- `mnpiSelfDeclared` を必須にする。
- `HypothesisMnpiSelfDeclarationUpdate`
- `mnpiSelfDeclared` の事前更新を許可し、昇格判定前に監査可能な形で保持する。

- `HypothesisDetail`
- `instrumentType`, `insiderRisk`, `mnpiSelfDeclared`, `autoPromotionEligible`, `promotionMode` を保持する。

- `ComplianceControls`
- `partnerRestrictedSymbols`, `autoPromotionEnabled` を保持する。

### 7.2 AsyncAPI

- `HypothesisDecisionPayload`
- `actionReasonCode`, `promotionMode`, `mnpiSelfDeclared`, `insiderRisk` を保持する。

## 8. 監査ログ必須項目

- `timestamp`
- `identifier`
- `trace`
- `user`
- `symbol`
- `instrumentType`
- `insiderRisk`
- `mnpiSelfDeclared`
- `promotionMode` (`manual` / `auto`)
- `actionReasonCode`
- `result` (`success` / `failed`)

## 9. エラーコード方針

- `COMPLIANCE_REVIEW_REQUIRED`
- `COMPLIANCE_MNPI_SUSPECTED`
- `COMPLIANCE_RESTRICTED_SYMBOL`
- `COMPLIANCE_BLACKOUT_ACTIVE`
- `OPERATION_NOT_ALLOWED`

## 10. テスト観点

1. 自動昇格正常系
- ETF + low + 自己申告true + 非ブロック銘柄 + demo30日以上で `auto` 昇格する。

2. 自動昇格拒否系
- `mnpiSelfDeclared=false` で自動昇格されない。
- `partnerRestrictedSymbols` 該当で自動昇格されない。

3. 手動昇格系
- 個別株は `POST /hypotheses/{identifier}/promote` のみで昇格できる。

4. 監査系
- 全昇格判定で `trace`, `actionReasonCode`, `mnpiSelfDeclared`, `promotionMode` が記録される。
