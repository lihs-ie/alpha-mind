# signal-generator 外部設計書

最終更新日: 2026-02-28

## 1. サービス概要

- 役割: 銘柄シグナル（売買候補スコア）を生成する。
- 主な責務: 特徴量と承認済みモデルを用いてスコアを計算し、下流へ配信する。
- 主な利用者: `portfolio-planner`。

## 2. 採用技術と比較

| 項目 | 採用 | 比較対象 | 採用理由 |
|---|---|---|---|
| モデル実行 | Python + LightGBM/scikit-learn | 深層学習中心 | 表形式データでの実装・運用コストが低い |
| 実行基盤 | Cloud Run Job | GPU常駐環境 | MVPではCPUバッチで十分 |
| モデル管理 | MLflow（外部管理） | 手作業管理 | モデル版と評価結果の追跡を標準化できる |

## 3. 外部インターフェース

### 3.1 購読イベント

- `features.generated`

### 3.2 発行イベント

- `signal.generated`
- `signal.generation.failed`

### 3.3 外部依存

- Cloud Storage（特徴量/推論結果）
- モデルレジストリ（MLflow）
- Firestore（最新シグナル要約）

## 4. ユースケース

### UC-SG-01: 日次シグナル生成

- 事前条件: 承認済みモデルが存在する。
- トリガー: `features.generated`受信。
- 成果: 銘柄スコアを保存し、`signal.generated`発行。

### UC-SG-02: モデル未承認時の停止

- 事前条件: 承認済みモデルが存在しない。
- トリガー: 生成要求受信。
- 成果: `signal.generation.failed`を発行し、注文作成を停止。

## 5. 非機能要件

- 完了目標: 1サイクル`10分以内`。
- 監査: `modelVersion`, `featureVersion`, `trace`を必須記録。
- 品質ゲート: モデルは外部検証基準を満たす版のみ利用可能。
- 追加品質ゲート: 評価時にコスト（手数料・スリッページ）控除後指標を必須とする。
- 劣化監視: 直近期間での成績低下が閾値を超えたモデルは候補運用へ差し戻す。

## 6. スコープ外

- モデル学習パイプラインの内部処理。
- ポートフォリオ最適化。
- 注文執行。
- AI提案戦略の無審査採用（必ずモデル検証と運用者承認を経る）。
