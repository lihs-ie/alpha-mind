# 内部設計書 作成ガイド

最終更新日: 2026-02-24

## 1. 目的

- 本ガイドは、外部設計を実装可能な粒度へ落とし込むための内部設計書作成ルールを定義する。
- 対象はイベント駆動マイクロサービス（BFF含む）。

## 2. 準拠する標準・フレームワーク

### 2.1 標準

1. IEEE 1016
- ソフトウェア設計記述（SDD）の構成基準として採用する。

2. ISO/IEC/IEEE 42010:2022
- 利害関係者・関心事・ビューで設計情報を整理する。

3. ISO/IEC 25010
- 品質特性（性能、可用性、保守性、セキュリティ等）の定義基準に使用する。

### 2.2 実務フレームワーク

1. arc42
- `Building Block View`（静的分解）
- `Runtime View`（実行時シナリオ）
- `Crosscutting Concepts`（横断方針）

2. C4 Model
- Container/Componentレベルで責務と依存を可視化する。

### 2.3 契約仕様

1. OpenAPI（同期API）
2. AsyncAPI（非同期イベント）
3. CloudEvents（イベント共通属性）
4. RFC 9457（エラー表現）
5. W3C Trace Context（トレース伝播）

## 3. 内部設計の記述対象

- サービス内処理ステップ
- モジュール分割（責務境界）
- データモデル（永続・一時・キャッシュ）
- 状態遷移（正常/異常）
- 冪等性・再試行・整合性
- 監視項目・アラート条件
- テスト方針
- Skill資産（skillVersion / instructionProfileVersion / failureKnowledge連携）

## 4. 記述しない対象

- 画面文言・UIレイアウト詳細
- クラス実装全文
- IaCやデプロイ手順の詳細

## 5. 作成手順（推奨）

1. サービス責務を固定する
- 1サービス1責務を明確化する。

2. 入出力契約を固定する
- APIはOpenAPI、イベントはAsyncAPI/CloudEventsで定義する。

3. 処理フローを設計する
- 正常系/異常系/再試行系を分離して記述する。

4. データ所有を定義する
- どのサービスがどのデータを正として保持するかを明記する。

5. 冪等性を設計する
- `identifier` で重複処理を防止する。

6. 品質要件を数値化する
- レイテンシ、成功率、DLQ許容値をSLOとして明示する。

7. 運用を設計する
- 監視、アラート、手動復旧手順を定義する。

8. 設計判断を記録する
- ADR（採用理由/非採用理由）を残す。

## 6. 内部設計書テンプレート

1. サービス概要（内部責務）
2. 処理フロー（正常/異常）
3. 内部モジュール設計
4. データモデル
5. エラー処理・再試行・冪等性
6. 監視・運用
7. テスト方針
8. 未決事項

## 7. イベント駆動マイクロサービス向け必須記載

### 7.1 イベント設計

- イベント名規則: `<domain>.<action>.<result>`
- 必須属性: `identifier`, `eventType`, `occurredAt`, `trace`, `schemaVersion`, `payload`
- バージョニング: `schemaVersion` で後方互換性を管理

### 7.2 再試行とDLQ

- 一時障害は指数バックオフで再試行
- 恒久障害はDLQへ退避
- `*.failed` イベントを発行し、運用に通知

### 7.3 整合性

- 分散トランザクションは避け、最終的整合性を前提に設計
- 状態更新とイベント発行の順序保証方針を明記

### 7.4 セキュリティ

- 外部公開はBFFのみ
- シークレットはSecret Managerで管理
- 監査ログに操作主体と`trace`を保存

## 8. 品質要件の書き方（本プロジェクト向け）

| 観点 | 最低記載事項 | 記載例 |
|---|---|---|
| 性能 | p95遅延、処理時間上限 | `signal.generated` 生成を10分以内 |
| 可用性 | 月間可用性目標 | 99.5%/月 |
| 信頼性 | 再試行回数、DLQ閾値 | 再試行3回、DLQ率<0.5% |
| 監査性 | 追跡キー、保持期間 | `trace`必須、1年保持 |
| セキュリティ | 認証・認可・秘密管理 | JWT + IAM + Secret Manager |

## 9. レビュー用チェックリスト

- 責務が単一か
- 同期/非同期契約が記載されているか
- 正常系/異常系が分離されているか
- 冪等性キーが定義されているか
- SLOが数値で定義されているか
- 復旧手順があるか
- 未決事項が明示されているか
