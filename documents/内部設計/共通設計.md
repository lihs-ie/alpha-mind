# 共通内部設計

最終更新日: 2026-02-28

## 1. 全体アーキテクチャ前提

- 実行基盤: Cloud Run
- イベントバス: Pub/Sub
- トランザクションデータ: Firestore
- 大容量履歴: Cloud Storage (Parquet)
- BFF: `bff`
- 追加サービス: `insight-collector`, `agent-orchestrator`, `hypothesis-lab`

## 2. イベントエンベロープ

全イベントは以下の共通属性を持つ。

| 項目 | 型 | 必須 | 説明 |
|---|---|---|---|
| identifier | string | 必須 | 冪等性キー |
| eventType | string | 必須 | イベント種別 |
| occurredAt | string | 必須 | ISO8601 UTC |
| trace | string | 必須 | 横断追跡ID |
| schemaVersion | string | 必須 | スキーマ版 |
| payload | object | 必須 | 業務データ |
| skillVersion | string | 任意 | Skill版（Skill実行イベント時） |
| instructionProfileVersion | string | 任意 | 指示書版（仮説生成時） |

## 3. Firestoreコレクション方針

| コレクション | 用途 | 主なキー |
|---|---|---|
| settings | 戦略設定、リスク設定 | `strategy`（固定ドキュメント） |
| operations | 稼働状態、kill switch | `runtime`（固定ドキュメント） |
| positions | 保有状態 | `symbol` |
| orders | 注文候補/承認/執行状態 | `identifier` |
| model_registry | モデル版メタ情報 | `modelVersion` |
| audit_logs | 監査イベント索引 | `identifier` |
| compliance_controls | 制限銘柄/ブラックアウト等のコンプライアンス制御 | `trading`（固定ドキュメント） |
| source_policies | 収集可能ソース・規約条件 | `identifier` |
| idempotency_keys | 処理済みイベント管理 | `identifier` |
| skill_registry | Skill定義管理 | `identifier` |
| code_reference_templates | 自作コード参照テンプレート管理 | `identifier` |
| insight_records | 定性インサイト保存 | `identifier` |
| hypothesis_registry | 仮説ライフサイクル管理 | `identifier` |
| backtest_runs | バックテスト結果 | `identifier` |
| demo_trade_runs | デモトレード結果 | `identifier` |
| failure_knowledge | 失敗知見 | `identifier` |
| instruction_profiles | Markdown指示書管理 | `identifier` |

## 4. エラーハンドリング共通方針

- 一時障害: 指数バックオフで最大3回再試行。
- 恒久障害: DLQへ転送し、`*.failed`イベントを発行。
- バリデーション違反: 再試行せず即時失敗扱い。

## 5. 冪等性共通方針

- 受信時に `idempotency_keys/{identifier}` を確認。
- 未処理なら処理開始、完了時に `processedAt` を記録。
- 既処理なら副作用を起こさず成功として終了。

## 6. 監視・可観測性

- すべてのログに `trace`, `identifier`, `service` を含める。
- 主要メトリクス:
- 処理成功率
- 処理遅延（p95）
- DLQ件数
- 外部APIエラー率
- インサイト根拠欠損率
- 仮説重複率

## 7. セキュリティ共通方針

- 外部APIキーは Secret Manager 管理。
- BFFのみをインターネット公開し、他サービスは内部通信に限定。
- 監査ログは最低1年保持（MVP暫定）。
